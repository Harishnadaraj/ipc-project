Project Overview — Comprehensive IPC Framework

This project demonstrates a unified IPC solution for local processes: FIFO, POSIX message queues, and shared memory — packaged with security, monitoring, and developer tools for real demonstrations and learning.

Goals

  • Provide a single API that abstracts multiple IPC transports.
  • Include security features (token-based access and POSIX permissions).
  • Offer monitoring & tools for practical visibility.

Expected Outcomes

  • A C library exposing create/open/send/receive/close operations.
  • Backends for FIFO, MQ, SHM with semaphores.
  • CLI utility and Python bindings for quick experimentation.

Scope

  • Local host IPC (not distributed messaging).
  • Educational & demonstrative — production hardening optional.
  • Cross-platform focus on POSIX systems (Linux recommended).

Quick API Example

// create channel ipc_channel_t *ch = ipc_create_channel(IPC_TYPE_SHM, "frames", &opts); // send / receive ipc_send(ch, buf, size); ipc_receive(ch, buf, max, &got);

Platform

POSIX-compliant systems; on Windows use WSL for best compatibility.

Why these transports?

  • FIFO — Easy to implement, suitable for streaming logs or pipes.
  • Message Queues — Structured messages, priorities, ideal for task queues.
  • Shared Memory — Highest throughput for large data via zero-copy.
