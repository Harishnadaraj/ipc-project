Functionalities

This page breaks down all the core functionalities that your IPC Framework supports. These functionalities help processes communicate securely, efficiently, and predictably.

1. FIFO (Named Pipes)

FIFO provides a simple, one-directional communication channel between processes.

Key Features:

  • Stream-based communication (bytes flow continuously)
  • Ideal for logging, process pipelines, and live data streams
  • Blocking and non-blocking modes supported
  • Simple read/write interface

Example Use Case:

A data producer writes logs to a FIFO, while a monitoring process continuously reads them.

2. POSIX Message Queues

Message Queues allow asynchronous communication using structured messages with optional priority levels.

Key Features:

  • Priority messaging (higher priority dequeued first)
  • Non-blocking send/receive options
  • Ideal for job scheduling, event handling, and worker systems
  • Reliable delivery with message boundaries preserved

Example Use Case:

Event-driven systems where multiple producers queue tasks for worker processes.

3. Shared Memory (With Semaphores)

Shared Memory is the fastest IPC method because processes read directly from a common memory region.

Key Features:

  • Zero-copy data sharing
  • Ideal for real-time systems or large binary data transfer
  • Requires synchronization with semaphores to avoid race conditions
  • High throughput and extremely low latency

Example Use Case:

A sensor generating high-frequency data that must be processed in real-time by another process.

4. Security Layer

The framework includes built-in security features to protect IPC channels from misuse.

Key Features:

  • Token-based authorization
  • Strict POSIX permissions (0600)
  • Optional AES-GCM message encryption
  • Access logging for auditing

Example Use Case:

Rejecting unauthorized processes attempting to access shared memory or message queues.

5. Monitoring & Metrics

The framework includes real-time observability options.

Key Features:

  • Message throughput tracking
  • Latency histograms
  • Queue depth monitoring
  • CLI tool (ipcctl) for inspecting channels

Example Use Case:

Detecting system bottlenecks, debugging communication delays, and performance tuning.
