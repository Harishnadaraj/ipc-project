Practical IPC Examples

Explore real-world scenarios demonstrating when and why to choose FIFO, Message Queues, or Shared Memory.

1. Real-time Camera Streaming (Shared Memory)

High throughput Zero copy Low latency

A camera process generates 1080p frames at 30fps. Copying each frame is expensive, so shared memory is used with a ring buffer.

  • Producer writes raw frames to shared memory.
  • Semaphore signals data availability.
  • Consumer reads frames without extra copies.

Code snippet:

int fd = shm_open("/frames", O_CREAT | O_RDWR, 0600); ftruncate(fd, FRAME_SIZE * 4); uint8_t* buf = mmap(NULL, size, PROT_READ | PROT_WRITE, MAP_SHARED, fd, 0); sem_post(frame_ready);

This technique is used in robotics, autonomous vehicles, and multimedia pipelines.

2. Task Scheduling (Message Queues)

Priority based Structured messages

A web front-end sends user tasks to a backend worker pool. Message queues help prioritize urgent tasks without modifying existing code.

  • Producer places structured jobs with priority metadata.
  • Consumer workers dequeue and process tasks in order.
  • Supports non-blocking receives for scalable queues.

Code snippet:

struct job j = { .task_id=101, .priority=3 }; mq_send(mq, (char*)&j, sizeof(j), j.priority);

Ideal for job dispatchers, rendering pipelines, and microservice task runners.

3. Log Aggregation (FIFO)

Simple Stream-based

Multiple child processes generate logs. Instead of writing to many files, all logs are streamed into a single FIFO.

  • Parent creates FIFO "/logs.pipe".
  • Children write logs to FIFO.
  • Aggregator process reads the combined stream.

Code snippet:

mkfifo("logs.pipe", 0666); int fd = open("logs.pipe", O_WRONLY); write(fd, "Child 1: Started\n", 18);

Used by system daemons, container runtimes, and debugging tools.

4. Multi-process Chat App (Hybrid IPC)

Hybrid model Real-time

A simple multi-process chat system uses:

  • Message Queue → Chat message routing
  • Shared Memory → Fast emoji/sticker transfers
  • FIFO → System logs and debug information

Hybrid IPC is extremely powerful because it uses each mechanism for what it does best.

5. Sensor Fusion System (Shared Memory + MQ)

Robotics AI Systems

Autonomous systems often gather data from many sensors:

  • Shared Memory → Send high-volume sensor frames
  • Message Queue → Send control commands / metadata

Fast + structured communication = optimal system performance.
